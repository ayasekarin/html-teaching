Sync / Async Workshop
今天怎么玩
  • 每个 Lab 都是一个“有坑的页面”（多数来自错误示例）。
  • 你们的目标：把它改成“可用 + 可观测 + 可维护”的异步实现。
  • 你可以先做同步版本，再改成异步版本，对比体验差异。
统一验收标准（所有 Lab 通用）
  1. 有 loading / error / empty 三态
  2. 请求可取消（AbortController）
  3. 超时处理（timeout）
  4. 失败可重试（retry）
  5. 日志可追踪（requestId + 时长）
公共函数（你只用这些）
入口页会把工具挂到 window.WS 上，大家在各 Lab 里直接调用。
